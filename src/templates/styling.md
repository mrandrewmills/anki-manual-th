# स्टाइलिंग और HTML

<!-- toc -->

## कार्ड स्टाइलिंग

आप YouTube पर [कार्ड स्टाइल करने के बारे में एक वीडियो](http://www.youtube.com/watch?v=F1j1Zx0mXME&yt:cc=on) देख सकते हैं। वीडियो Anki 2.0 का इंटरफ़ेस दिखाता है, लेकिन अवधारणाएँ काफी हद तक समान हैं।

कार्ड स्क्रीन का स्टाइलिंग अनुभाग "बैक टेम्पलेट" बटन के बगल में "स्टाइलिंग" बटन पर क्लिक करके एक्सेस किया जा सकता है। उस अनुभाग में, आप कार्ड का पृष्ठभूमि रंग, डिफ़ॉल्ट फ़ॉन्ट, पाठ संरेखण, और इसी तरह बदल सकते हैं।

आपके लिए उपलब्ध मानक विकल्प हैं:

**फ़ॉन्ट-फ़ैमिली**\ 
कार्ड पर उपयोग करने के लिए फ़ॉन्ट का नाम। यदि आपके फ़ॉन्ट में "MS Unicode" जैसे रिक्त स्थान हैं, तो आपको फ़ॉन्ट नाम को दोहरे उद्धरणों में घेरने की आवश्यकता है, जैसे `font-family: "MS Unicode";`। एक ही कार्ड पर कई फ़ॉन्ट का उपयोग करना भी संभव है; उस पर जानकारी के लिए, कृपया नीचे देखें।

**फ़ॉन्ट-साइज़**\ 
pixel में फ़ॉन्ट का आकार। इसे बदलते समय, सुनिश्चित करें कि आप अंत में px छोड़ दें।

**टेक्स्ट-एलाइन**\ 
क्या पाठ को केंद्र, बाएं, या दाएं संरेखित किया जाना चाहिए।

**रंग**\ 
पाठ का रंग। "ब्लू", "लाइटयेलो" जैसे साधारण रंग नाम काम करेंगे, या आप मनमाने रंग चुनने के लिए HTML रंग कोड का उपयोग कर सकते हैं। अधिक जानकारी के लिए कृपया [यह वेबपेज](https://htmlcolorcodes.com/) देखें।

**बैकग्राउंड-कलर**\ 
कार्ड पृष्ठभूमि का रंग।

कोई भी CSS स्टाइलिंग अनुभाग में रखा जा सकता है - उन्नत उपयोगकर्ता पृष्ठभूमि छवि या ग्रेडिएंट जोड़ने जैसी चीजें करना चाह सकते हैं, उदाहरण के लिए। यदि आप कुछ विशेष स्वरूपण प्राप्त करने के बारे में सोच रहे हैं, तो कृपया वेब पर CSS में इसे कैसे करना है, इस बारे में जानकारी के लिए खोज करें, क्योंकि बहुत सारे दस्तावेज़ उपलब्ध हैं।

स्टाइलिंग सभी कार्डों के बीच साझा की जाती है, जिसका अर्थ है कि जब आप एक समायोजन करते हैं तो यह उस नोट प्रकार के सभी कार्डों को प्रभावित करेगा। हालाँकि, कार्ड-विशिष्ट स्टाइलिंग निर्दिष्ट करना भी संभव है। निम्नलिखित उदाहरण सभी कार्डों पर पीली पृष्ठभूमि का उपयोग करेगा सिवाय पहले वाले के:

```css
.card {
  background-color: yellow;
}
.card1 {
  background-color: blue;
}
```

## छवि का आकार बदलना

Anki डिफ़ॉल्ट रूप से छवियों को स्क्रीन पर फिट करने के लिए छोटा करता है। आप अपने स्टाइलिंग अनुभाग के नीचे (डिफ़ॉल्ट `.card { ... }` के बाहर) निम्नलिखित जोड़कर इसे बदल सकते हैं:

```css
img {
  max-width: none;
  max-height: none;
}
```

AnkiDroid को कभी-कभी [स्क्रीन पर छवियों को स्केल करने में परेशानी](https://github.com/ankidroid/Anki-Android/issues/3612) होती है। सीएसएस का उपयोग करके अधिकतम छवि आयाम सेट करने से इसे ठीक किया जाना चाहिए, लेकिन AnkiDroid 2.9 के अनुसार इसे अनदेखा किया जाता है। एक फिक्स प्रत्येक स्टाइल निर्देश में `!important` जोड़ना है, उदाहरण के लिए:

```css
img {
  max-width: 300px !important;
  max-height: 300px !important;
}
```

यदि आप छवियों के लिए स्टाइल बदलने का प्रयास करते हैं और पाते हैं कि चिह्नित कार्डों पर दिखाई देने वाला तारा प्रभावित होता है (उदाहरण के लिए, यह बहुत बड़ा हो जाता है), तो आप इसे निम्नलिखित के साथ लक्षित कर सकते हैं:

```css
img#star {
  ...;
}
```

आप Chrome का उपयोग करके इंटरैक्टिव रूप से कार्डों की स्टाइलिंग का पता लगा सकते हैं:

<https://addon-docs.ankiweb.net/porting2.0.html#webview-changes>

Anki 2.1.50+ संपादक के भीतर छवि का आकार बदलने का मूल रूप से समर्थन करता है।

## फ़ील्ड स्टाइलिंग

डिफ़ॉल्ट स्टाइलिंग पूरे कार्ड पर लागू होती है। आप कुछ फ़ील्ड या कार्ड के हिस्से को एक अलग फ़ॉन्ट, रंग, आदि का उपयोग करने के लिए भी बना सकते हैं। यह विदेशी भाषाओं का अध्ययन करते समय विशेष रूप से महत्वपूर्ण है, क्योंकि Anki कभी-कभी वर्णों को सही ढंग से प्रदर्शित करने में असमर्थ होगा जब तक कि एक उपयुक्त फ़ॉन्ट का चयन नहीं किया जाता है।

मान लीजिए आपके पास एक "एक्सप्रेशन" फ़ील्ड है, और आप उसे macOS थाई फ़ॉन्ट "Ayuthaya" देना चाहते हैं। कल्पना कीजिए कि आपका टेम्पलेट पहले से ही पढ़ता है:

    What is {{Expression}}?

    {{Notes}}

हमें पाठ को स्टाइल करने के लिए HTML में लपेटने की आवश्यकता है। हम पाठ के सामने निम्नलिखित रखेंगे:

    <div class=mystyle1>

और इसके पीछे निम्नलिखित:

    </div>

पाठ को उपरोक्त की तरह लपेटकर, हम Anki को एक कस्टम स्टाइल "mystyle1" के साथ लपेटे हुए पाठ को स्टाइल करने के लिए कहते हैं, जिसे हम बाद में बनाएंगे।

इस प्रकार यदि हम पूरे "What is ...?" अभिव्यक्ति को थाई फ़ॉन्ट का उपयोग करना चाहते थे, तो हम उपयोग करेंगे:

    <div class=mystyle1>What is {{Expression}}?</div>

    {{Notes}}

और यदि हम केवल अभिव्यक्ति फ़ील्ड को थाई फ़ॉन्ट का उपयोग करना चाहते थे, तो हम उपयोग करेंगे:

    What is <div class=mystyle1>{{Expression}}</div>?

    {{Notes}}

टेम्पलेट संपादित करने के बाद, अब हमें टेम्पलेट के बीच स्टाइलिंग अनुभाग में जाने की आवश्यकता है। इसे संपादित करने से पहले, यह कुछ ऐसा दिखना चाहिए:

```css
.card {
  font-family: arial;
  font-size: 20px;
  text-align: center;
  color: black;
  background-color: white;
}
```

अपनी नई स्टाइल को नीचे जोड़ें, ताकि यह ऐसा दिखे:

```css
.card {
  font-family: arial;
  font-size: 20px;
  text-align: center;
  color: black;
  background-color: white;
}

.mystyle1 {
  font-family: ayuthaya;
}
```

आप स्टाइल में अपनी इच्छानुसार कोई भी स्टाइलिंग शामिल कर सकते हैं। यदि आप फ़ॉन्ट आकार भी बढ़ाना चाहते थे, तो आप mystyle1 अनुभाग को ऐसा दिखने के लिए बदलेंगे:

```css
.mystyle1 {
  font-family: ayuthaya;
  font-size: 30px;
}
```

अपने डेक के साथ कस्टम फ़ॉन्ट को बंडल करना भी संभव है, इसलिए आपको उन्हें अपने कंप्यूटर या मोबाइल डिवाइस पर स्थापित करने की आवश्यकता नहीं है। इस पर अधिक जानकारी के लिए कृपया [फ़ॉन्ट स्थापित करना](#installing-fonts) अनुभाग देखें।

## ऑडियो रीप्ले बटन

जब आपके कार्ड पर ऑडियो या टेक्स्ट टू स्पीच शामिल होता है, तो Anki बटन दिखाएगा जिन्हें आप ऑडियो को फिर से चलाने के लिए क्लिक कर सकते हैं।

यदि आप बटन नहीं देखना चाहते हैं, तो आप उन्हें वरीयताएँ स्क्रीन में छिपा सकते हैं।

आप अपने कार्ड स्टाइलिंग में उनकी उपस्थिति को अनुकूलित कर सकते हैं, उदाहरण के लिए, उन्हें छोटा और रंगीन बनाने के लिए, आप निम्नलिखित का उपयोग कर सकते हैं:

```css
.replay-button svg {
  width: 20px;
  height: 20px;
}
.replay-button svg circle {
  fill: blue;
}
.replay-button svg path {
  stroke: white;
  fill: green;
}
```

## टेक्स्ट डायरेक्शन

यदि आप दाएं से बाएं लिखी जाने वाली भाषा का उपयोग करते हैं, जैसे अरबी या हिब्रू, तो आप समीक्षा के दौरान सही प्रदर्शन के लिए `.card` अनुभाग में CSS `direction` प्रॉपर्टी जोड़ सकते हैं:

```css
.card {
  direction: rtl;
}
```

यह पूरे कार्ड की दिशा बदल देगा। आप कुछ HTML में उनके संदर्भों को लपेटकर केवल कुछ फ़ील्ड की दिशा बदल सकते हैं:

    <div dir="rtl">{{Front}}</div>

संपादक में फ़ील्ड की दिशा बदलने के लिए, कृपया [संपादन](../editing.md#customizing-fields) अनुभाग देखें।

## अन्य HTML

आपके टेम्पलेट में मनमाना HTML हो सकता है, जिसका अर्थ है कि इंटरनेट वेब पेजों पर उपयोग की जाने वाली सभी लेआउट संभावनाएं आपके कार्ड पर भी उपयोग की जा सकती हैं। तालिकाएँ, सूचियाँ, छवियाँ, बाहरी पृष्ठों के लिंक और इसी तरह सभी समर्थित हैं। उदाहरण के लिए तालिकाओं के साथ, आप लेआउट को बदल सकते हैं ताकि कार्ड का सामने और पीछे ऊपर और नीचे के बजाय बाएं और दाएं दिखाई दें।

HTML की सभी सुविधाओं को कवर करना इस नियमावली के दायरे से बाहर है, लेकिन वेब पर HTML के लिए बहुत सारे अच्छे परिचयात्मक गाइड उपलब्ध हैं यदि आप अधिक जानना चाहते हैं।

## ब्राउज़र उपस्थिति

यदि आपके कार्ड टेम्पलेट जटिल हैं, तो [कार्ड सूची](../browsing.md#cardnote-table) में प्रश्न और उत्तर कॉलम ("फ्रंट" और "बैक" नामक) को पढ़ना मुश्किल हो सकता है। "ब्राउज़र उपस्थिति" विकल्प आपको केवल ब्राउज़र में उपयोग किए जाने वाले एक कस्टम टेम्पलेट को परिभाषित करने की अनुमति देता है, ताकि आप केवल महत्वपूर्ण फ़ील्ड शामिल कर सकें और यदि आप चाहें तो क्रम बदल सकें। सिंटैक्स मानक कार्ड टेम्पलेट के समान है।

इस विकल्प का उपयोग करते समय, यदि पाठ प्रश्न कॉलम में उत्तर कॉलम की शुरुआत में दोहराया जाता है, तो Anki पाठ को केवल प्रश्न कॉलम में प्रदर्शित करेगा। उदाहरण के लिए, यदि प्रश्न कॉलम पाठ "लद्दाख में लोग बोलते हैं", और उत्तर "लद्दाख में लोग लद्दाखी बोलते हैं", तो उत्तर कॉलम केवल "लद्दाखी" प्रदर्शित करेगा, बाकी को छोड़कर।

## प्लेटफ़ॉर्म-विशिष्ट CSS

Anki कुछ विशेष CSS कक्षाएं परिभाषित करता है जो आपको विभिन्न प्लेटफार्मों के लिए अलग-अलग स्टाइलिंग को परिभाषित करने की अनुमति देती हैं। नीचे दिया गया उदाहरण दिखाता है कि आप जहां समीक्षा कर रहे हैं, उसके आधार पर फ़ॉन्ट को कैसे भिन्न कर सकते हैं:

```css
/* विंडोज */
.win .example {
  font-family: "Example1";
}
/* macOS */
.mac .example {
  font-family: "Example2";
}
/* लिनक्स डेस्कटॉप */
.linux:not(.android) .example {
  font-family: "Example3";
}
/* लिनक्स डेस्कटॉप और एंड्रॉइड डिवाइस दोनों */
.linux .example {
  font-family: "Example4";
}
/* एंड्रॉइड और iOS दोनों */
.mobile .example {
  font-family: "Example5";
}
/* iOS */
.iphone .example,
.ipad .example {
  font-family: "Example6";
}
/* एंड्रॉइड */
.android .example {
  font-family: "Example7";
}
```

और टेम्पलेट में:

```html
<div class="example">{{Field}}</div>
```

आप AnkiWeb का उपयोग करते समय कुछ ब्राउज़र का चयन करने के लिए .gecko, .opera, और .ie जैसी प्रॉपर्टी का भी उपयोग कर सकते हैं। विकल्पों की पूरी सूची के लिए कृपया <http://rafael.adm.br/css_browser_selector/> देखें।

## फ़ॉन्ट स्थापित करना

आप सीधे Anki में फ़ॉन्ट स्थापित कर सकते हैं। यह तब उपयोगी होता है जब आप एक कार्य या स्कूल कंप्यूटर पर Anki का उपयोग कर रहे हों जहाँ आपके पास नए फ़ॉन्ट स्थापित करने की अनुमति नहीं है, या यदि आप एक मोबाइल डिवाइस पर Anki का उपयोग कर रहे हैं।

Anki सबसे व्यापक रूप से उपयोग किए जाने वाले फ़ॉन्ट प्रारूपों का समर्थन करता है, जैसे ट्रू टाइप (.ttf), ओपन टाइप (.otf), वेब ओपन फ़ॉन्ट फ़ॉर्मेट (.woff) और अन्य।

### मीडिया फ़ोल्डर में फ़ॉन्ट जोड़ें

एक बार जब आप एक समर्थित फ़ॉन्ट डाउनलोड कर लेते हैं, जैसे "Arial.ttf", तो आपको इसे मीडिया फ़ोल्डर में जोड़ना होगा।

1. फ़ाइल का नाम बदलें, शुरुआत में एक अंडरस्कोर जोड़ें, ताकि यह "_arial.ttf" जैसा हो जाए। एक अंडरस्कोर Anki को बताएगा कि यह फ़ाइल एक टेम्पलेट द्वारा उपयोग की जाएगी, और अप्रयुक्त मीडिया की जाँच करते समय इसे नहीं हटाया जाना चाहिए।

2. अपने कंप्यूटर के फ़ाइल ब्राउज़र में, अपने Anki एप्लिकेशन डेटा फ़ोल्डर में जाएं ([फ़ाइल स्थान](../files.md#file-locations) के विवरण के लिए देखें), और फिर अपने प्रोफ़ाइल फ़ोल्डर में (उदाहरण के लिए, "उपयोगकर्ता 1")।

3. फ़ोल्डर के अंदर, आपको collection.media नामक एक फ़ोल्डर दिखाई देना चाहिए। renamed फ़ाइल को उस फ़ोल्डर में खींचें।

### उस फ़ॉन्ट का उपयोग करने के लिए टेम्पलेट अपडेट करें

एक बार फ़ॉन्ट को मीडिया फ़ोल्डर में जोड़ दिया गया है, तो आपको टेम्पलेट को अपडेट करने की आवश्यकता है।

1. मुख्य स्क्रीन के शीर्ष पर **जोड़ें** पर क्लिक करें, और फिर शीर्ष बाएं बटन के साथ वह नोट प्रकार चुनें जिसे आप बदलना चाहते हैं।

2. **कार्ड** पर क्लिक करें।

3. स्टाइलिंग अनुभाग में, निम्नलिखित टेक्स्ट को नीचे जोड़ें (अंतिम "}" वर्ण के बाद), "_arial.ttf" को उस फ़ाइल के नाम से बदलें जिसे आपने अपने मीडिया फ़ोल्डर में कॉपी किया था:

```css
@font-face {
  font-family: myfont;
  src: url("_arial.ttf");
}
```

केवल "arial" भाग बदलें, "myfont" भाग नहीं।

उसके बाद, आप या तो पूरे कार्ड के लिए फ़ॉन्ट बदल सकते हैं, या व्यक्तिगत फ़ील्ड के लिए। पूरे कार्ड के लिए फ़ॉन्ट बदलने के लिए, बस `.card` अनुभाग में `font-family:` लाइन का पता लगाएं और फ़ॉन्ट को "myfont" में बदल दें। केवल कुछ फ़ील्ड के लिए फ़ॉन्ट बदलने के लिए, कृपया ऊपर दिए गए [फ़ील्ड स्टाइलिंग](#field-styling) निर्देशों को देखें।

कृपया सुनिश्चित करें कि फ़ाइल नाम बिल्कुल मेल खाते हैं। यदि फ़ाइल arial.TTF है और आप arial.ttf इन अपने कार्ड टेम्पलेट में लिखते हैं, तो यह काम नहीं करेगा।

## नाइट मोड

आप वरीयताएँ स्क्रीन में नाइट मोड सक्षम होने पर टेम्पलेट कैसे दिखाई देते हैं, इसे अनुकूलित कर सकते हैं।

यदि आप एक हल्का ग्रे बैकग्राउंड चाहते थे, तो आप कुछ इस तरह का उपयोग कर सकते थे:

```css
.card.nightMode {
  background-color: #555;
}
```

यदि आपके पास एक "myclass" स्टाइल है, तो नाइट मोड सक्षम होने पर निम्नलिखित टेक्स्ट को पीले रंग में दिखाएगा:

```css
.nightMode .myclass {
  color: yellow;
}
```

## फेडिंग और स्क्रॉलिंग

Anki डिफ़ॉल्ट रूप से उत्तर पर स्वचालित रूप से स्क्रॉल करेगा। यह `id=answer` के साथ एक HTML तत्व की तलाश करता है, और उस पर स्क्रॉल करता है। आप स्क्रॉलिंग स्थिति को समायोजित करने के लिए `id` को एक अलग तत्व पर रख सकते हैं, या स्क्रॉलिंग बंद करने के लिए `id=answer` हटा सकते हैं।

एक कार्ड का प्रश्न पक्ष डिफ़ॉल्ट रूप से फेड होता है। यदि आप इस देरी को समायोजित करना चाहते हैं, तो आप अपने फ्रंट कार्ड टेम्पलेट के शीर्ष पर निम्नलिखित रख सकते हैं:

```html
<script>
  qFade = 100;
  if (typeof anki !== "undefined") anki.qFade = qFade;
</script>
```

100 (मिलीसेकंड) डिफ़ॉल्ट है; फेडिंग को अक्षम करने के लिए 0 पर सेट करें।

## जावास्क्रिप्ट

चूंकि Anki कार्ड वेब पेजों की तरह माने जाते हैं, इसलिए कार्ड टेम्पलेट के माध्यम से आपके कार्ड पर कुछ जावास्क्रिप्ट एम्बेड करना संभव है। एक अच्छे संदर्भ के लिए कृपया फ़ोरम में [यह पोस्ट](https://forums.ankiweb.net/t/card-templates-user-input-101-buttons-keyboard-shortcuts-etc-guide/13756) पढ़ें।

क्योंकि जावास्क्रिप्ट एक उन्नत सुविधा है और बहुत सी चीजें गलत हो सकती हैं, **जावास्क्रिप्ट कार्यक्षमता बिना किसी समर्थन या वारंटी के प्रदान की जाती है**। हम जावास्क्रिप्ट लिखने में कोई सहायता प्रदान नहीं कर सकते हैं, और यह गारंटी नहीं दे सकते कि आपके द्वारा लिखा गया कोई भी कोड भविष्य के Anki अपडेट में संशोधन के बिना काम करता रहेगा। यदि आप अपने दम पर आने वाली किसी भी समस्या को हल करने में सहज नहीं हैं, तो कृपया जावास्क्रिप्ट का उपयोग करने से बचें।

प्रत्येक Anki क्लाइंट कार्ड डिस्प्ले को अलग तरह से लागू कर सकता है, इसलिए आपको प्लेटफार्मों पर व्यवहार का परीक्षण करने की आवश्यकता होगी। कई क्लाइंट एक लंबे समय तक चलने वाले वेबपेज को रखकर और कार्ड की समीक्षा किए जाने पर उसके हिस्सों को गतिशील रूप से अपडेट करके लागू किए जाते हैं, इसलिए आपके जावास्क्रिप्ट को document.getElementById() जैसी चीजों का उपयोग करके दस्तावेज़ के अनुभागों को अपडेट करने की आवश्यकता होगी, बजाय document.write() जैसी चीजों को करने के।

window.alert जैसे फ़ंक्शन उपलब्ध नहीं हो सकते हैं। Anki जावास्क्रिप्ट त्रुटियों को टर्मिनल पर लिखेगा, इसलिए आपको उन्हें देखने के लिए [कंसोल देखना होगा](https://addon-docs.ankiweb.net/console-output.html#console-output)। जावास्क्रिप्ट के साथ मुद्दों को डिबग करने के लिए, आप Chrome के [निरीक्षक](https://addon-docs.ankiweb.net/debugging.html#webviews) का उपयोग कर सकते हैं।
