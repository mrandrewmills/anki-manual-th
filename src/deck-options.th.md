# ตัวเลือกสำรับการ์ด (Deck Options)

<!-- toc -->

ตัวเลือกสำรับการ์ดควบคุมวิธีที่ Anki จัดตารางการ์ดเป็นหลัก ขอแนะนำให้คุณใช้เวลาสองสามสัปดาห์กับค่าปริยายเพื่อทำความเข้าใจว่า Anki ทำงานอย่างไร ก่อนที่จะเริ่มปรับแต่งตัวเลือก โปรดตรวจสอบให้แน่ใจว่าคุณเข้าใจตัวเลือกเหล่านี้ก่อนเปลี่ยนแปลง เพราะความผิดพลาดอาจลดประสิทธิภาพของ Anki ได้

บนคอมพิวเตอร์ของคุณ ให้ทำสิ่งใดสิ่งหนึ่งต่อไปนี้เพื่อเปิดตัวเลือกสำรับการ์ด:

- คลิกไอคอนรูปเฟืองบนหน้าจอ Decks
- เลือกสำรับบนหน้าจอ Decks แล้วคลิก **Options** ที่ด้านล่างของหน้าจอ
- คลิก **More > Options** ขณะอยู่ในโหมดทบทวน
- กด <kbd>O</kbd> ขณะอยู่ในโหมดทบทวน

นี่คือลิงก์โพสต์จากชุมชนเกี่ยวกับตัวเลือกสำรับการ์ดที่เคยมีการแบ่งปันไว้:

- [ตัวเลือกสำรับการ์ดอธิบายไว้](https://forums.ankiweb.net/t/deck-options-explained/213)
- [ตัวเลือกสำรับการ์ดในรูปแบบแผนที่ความคิด](https://forums.ankiweb.net/t/deck-options-in-a-mental-map/15757)

## ค่าปริยาย (Presets)

Anki อนุญาตให้คุณแชร์ตัวเลือกระหว่างสำรับต่าง ๆ เพื่อให้ง่ายต่อการอัปเดตตัวเลือกในหลาย ๆ สำรับพร้อมกัน โดยตัวเลือกจะถูกจัดกลุ่มในรูปแบบ _ค่าปริยาย_ (presets) หากคุณเปลี่ยนตัวเลือกในค่าปริยายใด การเปลี่ยนแปลงจะถูกใช้กับสำรับทั้งหมดที่ใช้ค่าปริยายเดียวกัน สำรับใหม่ทั้งหมดจะใช้ค่าปริยายชื่อ "Default"

หากต้องการเปลี่ยนตัวเลือกในสำรับใดสำรับหนึ่งโดยไม่กระทบสำรับอื่น ให้คลิกไอคอนลูกศรที่มุมขวาบนของหน้าต่างตัวเลือกสำรับ คุณจะสามารถทำสิ่งต่อไปนี้ได้:

- **บันทึก (Save)**: บันทึกการแก้ไขทั้งหมดที่คุณทำในตัวเลือกสำรับการ์ด
- **เพิ่มค่าปริยาย (Add Preset)**: เพิ่มค่าปริยายใหม่สำหรับสำรับนี้ โดยใช้ตัวเลือกปริยายเริ่มต้น
- **โคลน (Clone)**: ทำสำเนาค่าปริยายปัจจุบันของคุณ เหมาะสำหรับการแก้ไขบางตัวเลือกโดยเก็บตัวเลือกที่เหลือไว้เหมือนเดิม
- **เปลี่ยนชื่อ (Rename)**: เปลี่ยนชื่อค่าปริยายปัจจุบัน
- **ลบ (Delete)**: ลบค่าปริยายปัจจุบัน ซึ่งจะทำให้การซิงค์ครั้งถัดไปเป็นแบบ [ซิงค์ทางเดียว](./syncing.md#conflicts)
- **บันทึกให้กับสำรับย่อยทั้งหมด (Save to All Subdecks)**: เหมือนกับ **บันทึก** แต่ยังกำหนดค่าปริยายที่เลือกให้กับสำรับย่อยทั้งหมดของสำรับที่เลือกอยู่ในปัจจุบันด้วย

ตัวเลือกสำรับการ์ดจะไม่มีผลย้อนหลัง เช่น หากคุณเปลี่ยนตัวเลือกที่ควบคุมเวลาหน่วงหลังจากการ์ดล้มเหลว การ์ดที่คุณล้มเหลวก่อนเปลี่ยนตัวเลือกนี้จะยังคงใช้เวลาหน่วงเดิม ไม่ใช่เวลาหน่วงใหม่

## สำรับย่อย (Subdecks)

ถ้าสำรับของคุณมีสำรับย่อย และคุณต้องการให้สำรับย่อยหนึ่งหรือมากกว่านั้นมีการตั้งค่าต่างจากสำรับหลัก คุณสามารถกำหนดให้สำรับย่อยเหล่านั้นใช้ค่าปริยายที่ต่างกันได้  
เมื่อ Anki แสดงการ์ด จะตรวจสอบว่าสมาชิกการ์ดนั้นอยู่ในสำรับย่อยใด และใช้ตัวเลือกของสำรับนั้น มีข้อยกเว้นสองกรณี:

- **จำนวนการ์ดใหม่/วัน** และ **จำนวนการ์ดทบทวนสูงสุด/วัน** [ข้อจำกัด](#daily-limits) ของสำรับย่อย จะมีผลต่อจำนวนการ์ดที่สามารถดึงมาจากสำรับย่อยนั้นได้ แต่จำนวนรวมของการ์ดที่คุณเห็นในช่วงเรียนจะถูกควบคุมโดยข้อจำกัดของสำรับที่คุณเลือกเรียน
- ตัวเลือก [ลำดับการแสดงผล](#display-order) จะมาจากสำรับที่คุณเลือกเรียน ไม่ใช่สำรับของการ์ดปัจจุบัน

ตัวอย่าง สมมติว่าคุณมีชุดสำรับดังนี้:

    - สำรับ A (ค่าปริยาย 1)
      - สำรับ A::สำรับย่อย B (ค่าปริยาย 2)

ค่าปริยาย 1 และ ค่าปริยาย 2 เหมือนกันทั้งหมด ยกเว้นสองข้อดังนี้:

- ค่าปริยาย 1:
  - **ขั้นตอนการเรียนรู้ (Learning steps)**: `1m 10m`
  - **ลำดับใหม่/ทบทวน (New/review order)**: `Mix with reviews`
- ค่าปริยาย 2:
  - **ขั้นตอนการเรียนรู้ (Learning steps)**: `20m 2h`
  - **ลำดับใหม่/ทบทวน (New/review order)**: `Show after reviews`

ถ้าคุณเลือกเรียนสำรับย่อย B:

- ขั้นตอนการเรียนรู้สำหรับการ์ดใหม่ทั้งหมดจะเป็น `20m 2h` (ใช้ค่าปริยาย 2)
- การ์ดใหม่ทั้งหมดจะแสดงหลังการทบทวน (ใช้ค่าปริยาย 2)

ถ้าคุณเลือกเรียนสำรับ A:

- ขั้นตอนการเรียนรู้สำหรับการ์ดใหม่ในสำรับ A จะเป็น `1m 10m` (ใช้ค่าปริยาย 1)
- ขั้นตอนการเรียนรู้สำหรับการ์ดใหม่ในสำรับย่อย B จะเป็น `20m 2h` (ใช้ค่าปริยาย 2)
- การ์ดใหม่ทั้งหมดจะถูกผสมกับการทบทวน (ใช้ค่าปริยาย 1)

## ข้อจำกัดรายวัน (Daily Limits)

### การ์ดใหม่/วัน (New Cards/Day)

ตัวเลือกนี้ควบคุมจำนวนการ์ดใหม่ที่สามารถเพิ่มเข้ามาในแต่ละวันที่คุณเปิดโปรแกรม  
ถ้าคุณเรียนการ์ดใหม่ต่ำกว่าข้อจำกัด หรือพลาดวันเรียน ในวันถัดไป จำนวนจะถูกตั้งกลับไปตามค่าปริยายเดิม: คุณจะไม่ได้รับการ์ดเกินกว่าข้อจำกัดที่ตั้งไว้

เมื่อเรียนสำรับที่มีสำรับย่อย ข้อจำกัดที่ตั้งไว้ในแต่ละสำรับย่อยจะควบคุมจำนวนการ์ดสูงสุดที่ดึงมาจากสำรับนั้น ๆ  
แต่ข้อจำกัดของสำรับที่เลือกเรียนจะควบคุมจำนวนรวมของการ์ดทั้งหมดที่จะถูกแสดง

สำหรับเวอร์ชันก่อนหน้า ดูได้ที่ [หน้าคำถามที่พบบ่อยนี้](https://faqs.ankiweb.net/the-anki-2.1-scheduler.html)

การเรียนการ์ดใหม่จะทำให้จำนวนการทบทวนที่คุณต้องทำในแต่ละวันเพิ่มขึ้นชั่วคราว  
เพราะเนื้อหาที่เรียนใหม่ต้องถูกทบทวนหลายครั้งก่อนที่ระยะเวลาระหว่างการทบทวนจะเพิ่มขึ้นอย่างมีนัยสำคัญ  
ถ้าคุณเรียนการ์ดใหม่วันละประมาณ 20 ใบอย่างสม่ำเสมอ คุณสามารถคาดหวังได้ว่าการทบทวนรายวันจะอยู่ราว ๆ 200 ใบต่อวัน  
คุณสามารถลดจำนวนการทบทวนที่ต้องทำได้โดยลดจำนวนการ์ดใหม่ที่เพิ่มเข้ามาในแต่ละวัน จนกว่าภาระการทบทวนจะลดลง  
ผู้ใช้ Anki หลายคนเคยตื่นเต้นเรียนการ์ดใหม่หลายร้อยใบในช่วงวันแรกของการใช้โปรแกรม  
แต่หลังจากนั้นก็รู้สึกว่าการทบทวนที่ต้องทำมากเกินไปจนท่วมท้น

### การทบทวนสูงสุด/วัน (Maximum Reviews/Day)

อนุญาตให้คุณตั้งขีดจำกัดสูงสุดของจำนวนการ์ดทบทวนที่จะถูกแสดงในแต่ละวัน  
เมื่อถึงขีดจำกัดนี้แล้ว Anki จะไม่แสดงการ์ดทบทวนเพิ่มเติมในวันนั้น แม้ว่าจะมีการ์ดที่รอทบทวนอยู่ก็ตาม  
ถ้าคุณเรียนอย่างสม่ำเสมอ การตั้งค่านี้ช่วยลดความผันผวนของจำนวนการ์ดที่ถึงเวลาทบทวนในบางครั้ง และช่วยป้องกันไม่ให้คุณตกใจเมื่อกลับมาใช้ Anki หลังหยุดไปหลายวัน  
เมื่อการ์ดทบทวนถูกซ่อนไว้เนื่องจากตัวเลือกนี้ จะมีข้อความแสดงขึ้นในหน้าจอแสดงความยินดี แนะนำให้คุณพิจารณาเพิ่มขีดจำกัดหากมีเวลาว่าง

เมื่อเรียนสำรับที่มีสำรับย่อย ขีดจำกัดการทบทวนจะทำงานคล้ายกับขีดจำกัดการ์ดใหม่

Anki นับรวมการ์ดที่อยู่ในช่วง [ข้ามวัน](./preferences.md#scheduler) (การ์ดเรียนรู้ข้ามวัน) ในจำนวนการทบทวนด้วย  
ดังนั้นการ์ดเรียนรู้เหล่านั้นจะอยู่ภายใต้ขีดจำกัดการทบทวน

### ขีดจำกัดรายวันแยกตามสำรับ (Per-Deck Daily Limits)

สามารถใช้ค่าปริยายชุดเดียวกันกับสำรับต่าง ๆ พร้อมตั้งค่าขีดจำกัดเฉพาะสำหรับแต่ละสำรับได้  
ซึ่งช่วยให้ไม่ต้องสร้างค่าปริยายโคลนหลายชุด และทำให้ง่ายต่อการตั้งขีดจำกัดเฉพาะสำหรับสำรับย่อย

Anki มีตัวเลือกสามแบบสำหรับขีดจำกัดรายวัน:

- **ค่าปริยาย (Preset)**: ใช้กับสำรับทั้งหมดที่ใช้ค่าปริยายชุดนั้น
- **สำรับนี้ (This deck)**: เฉพาะกับสำรับใดสำรับหนึ่ง
- **เฉพาะวันนี้ (Today only)**: เฉพาะกับสำรับใดสำรับหนึ่ง และเป็นการตั้งชั่วคราว

### การ์ดใหม่ไม่สนใจขีดจำกัดการทบทวน (New Cards Ignore Review Limit)

โดยปริยาย ขีดจำกัดการทบทวนจะใช้กับการ์ดใหม่ด้วย และจะไม่แสดงการ์ดใหม่เมื่อถึงขีดจำกัดการทบทวน  
ถ้าเปิดใช้งานตัวเลือกนี้ การ์ดใหม่จะถูกแสดงไม่ว่าขีดจำกัดการทบทวนจะถึงหรือไม่ก็ตาม

หากคุณมีการ์ดทบทวนค้างจำนวนมาก แนะนำให้หยุดเพิ่มการ์ดใหม่จนกว่าจะตามทบทวนการ์ดค้างนั้นทัน  
เพราะถ้าเพิ่มการ์ดใหม่ในขณะที่ยังมีการ์ดทบทวนค้าง อาจทำให้ภาระการทบทวนหนักขึ้น

### ขีดจำกัดเริ่มจากสำรับหลัก (Limits Start From The Top)

โดยปกติ ขีดจำกัดรายวันของสำรับระดับบนจะไม่ถูกนำมาใช้ถ้าคุณเลือกเรียนสำรับย่อย  
เช่น สำรับหลักอาจมีขีดจำกัดการ์ดใหม่ 10 ใบ/วัน และสำรับย่อยมีขีดจำกัด 20 ใบ/วัน  
ขีดจำกัดที่ตั้งในสำรับหลักจะไม่ส่งผลต่อจำนวนการ์ดใหม่ที่คุณเรียนจากสำรับย่อยนั้น

เมื่อเปิดใช้งานตัวเลือกนี้ ขีดจำกัดที่ตั้งในสำรับระดับบนจะถูกบังคับใช้กับสำรับย่อยเมื่อเลือกเรียนสำรับย่อย  
ในตัวอย่างก่อนหน้า คุณจะสามารถเรียนการ์ดใหม่จากสำรับย่อยได้เพียง 10 ใบแทนที่จะเป็น 20 ใบ

ตัวเลือกนี้เหมาะสำหรับผู้ที่ต้องการเรียนสำรับย่อยแต่ต้องการจำกัดจำนวนการ์ดรวมของสำรับย่อยทั้งหมด

## การ์ดใหม่ (New Cards)

ตัวเลือกในส่วนนี้มีผลเฉพาะกับการ์ดใหม่และ  
[การ์ดเรียนรู้](getting-started.md#card-states) เท่านั้น  
เมื่อการ์ดผ่านการเรียนรู้ครบขั้นตอน (หมายถึงผ่านทุกขั้นตอนการเรียนรู้) ตัวเลือกในส่วนนี้จะไม่ส่งผลกับการ์ดนั้นอีกต่อไป

### ขั้นตอนการเรียนรู้ (Learning Steps)

ควบคุมจำนวนรอบการทบทวนและระยะเวลาหน่วงระหว่างรอบเหล่านั้น  
ต้องใส่เวลาหน่วงอย่างน้อยหนึ่งค่าขึ้นไป โดยคั่นด้วยช่องว่าง  
ทุกครั้งที่คุณคลิก **Good** ขณะทบทวน การ์ดจะเลื่อนไปยังขั้นตอนถัดไป  
และทุกครั้งที่คลิก **Again** การ์ดจะกลับไปยังขั้นตอนแรก

ตัวอย่าง สมมติว่าขั้นตอนการเรียนรู้ของคุณคือ `1m 10m 1d`

- เมื่อคุณคลิก **Again** การ์ดจะเริ่มที่ขั้นตอนแรก และจะแสดงอีกครั้งใน 1 นาทีถัดมา
- เมื่อคุณคลิก **Good** บนการ์ดใหม่ หรือหลังจากขั้นตอน 1 นาที การ์ดจะเลื่อนไปขั้นตอนถัดไป และจะแสดงอีกครั้งใน 10 นาที
- เมื่อคุณคลิก **Good** บนการ์ดหลังขั้นตอน 10 นาที การ์ดจะถูกหน่วงเวลาไปจนถึงวันถัดไป
- เมื่อคุณคลิก **Good** ในวันถัดไป การ์ดจะผ่านการเรียนรู้ (graduate) และกลายเป็นการ์ดทบทวน  
  การ์ดจะถูกแสดงอีกครั้งหลังจากระยะเวลาหน่วงที่ตั้งไว้ใน _graduating interval_

ปุ่ม **Hard** ทำงานแตกต่างกันขึ้นอยู่กับขั้นตอนที่คุณอยู่

- เมื่ออยู่ในขั้นตอนแรก ปุ่ม **Hard** จะแสดงการหน่วงเวลา `6m` ซึ่งเป็นค่าเฉลี่ยของสองขั้นแรก คือ `1m` กับ `10m`
  - ข้อยกเว้น: ถ้ามีเพียงขั้นตอนเดียว ปุ่ม **Hard** จะหน่วงเวลา 1.5 เท่าของขั้นตอนนั้น โดยเวลาหน่วงสูงสุดจะยาวกว่าเวลาขั้นตอนนั้นไม่เกิน 1 วัน
- เมื่ออยู่ในขั้นตอนอื่น ๆ ปุ่ม **Hard** จะทำให้ขั้นตอนนั้นถูกทำซ้ำอีกครั้ง

ถ้าไม่มีการ์ดอื่นให้เรียน Anki จะเริ่มแสดงการ์ดเรียนรู้ล่วงหน้าได้สูงสุด 20 นาทีตามค่าเริ่มต้น  
ถ้าต้องการปิดหรือเปลี่ยนเวลาล่วงหน้า ดูได้ที่ [การตั้งค่า](preferences.md)

#### ขอบเขตวัน (Day Boundaries)

Anki จัดการขั้นตอนสั้น ๆ กับขั้นตอนที่ [ข้ามขอบเขตวัน](./preferences.md#review) แตกต่างกัน  
กับขั้นตอนสั้น ๆ การ์ดจะถูกแสดงทันทีที่เวลาหน่วงผ่านไป โดยให้ความสำคัญกับการ์ดทบทวนและการ์ดใหม่  
เพื่อให้คุณตอบการ์ดได้ใกล้เคียงกับเวลาที่ตั้งไว้มากที่สุด  
แต่ถ้าขั้นตอนนั้นข้ามขอบเขตวัน เวลาหน่วงจะถูกแปลงเป็นจำนวนวันโดยอัตโนมัติ  
เช่น ถ้าวันใหม่เริ่มหลัง 5 ชั่วโมง และเวลาหน่วงคือ 6 ชั่วโมง Anki จะเปลี่ยนเวลาหน่วงเป็น 1 วัน

### ระยะเวลาหลังผ่านการเรียนรู้ (Graduating Interval)

จำนวนวันต้องรอก่อนจะแสดงการ์ดอีกครั้ง หลังจากกดปุ่ม Good ในขั้นตอนสุดท้ายของการเรียนรู้  
หมายความว่านี่คือระยะเวลาหน่วงแรกหลังจากการ์ดเรียนรู้ผ่านการเรียนรู้ (graduate)  
ดูตัวอย่างได้จาก [ส่วนก่อนหน้านี้](deck-options.md#learning-steps)

### ระยะเวลาแบบง่าย (Easy Interval)

จำนวนวันที่ต้องรอก่อนจะแสดงการ์ดอีกครั้ง หลังจากกดปุ่ม Easy บนการ์ดนั้น

ปุ่ม **Easy** จะเปลี่ยนการ์ดเรียนรู้ให้กลายเป็นการ์ดทบทวนไม่ว่าจะอยู่ในขั้นตอนใด และกำหนดเวลาหน่วงตามที่ตั้งไว้ในตัวเลือกนี้  
ระยะเวลาแบบง่ายควรมีค่าไม่น้อยกว่าระยะเวลาหลังผ่านการเรียนรู้ (graduating interval) และโดยทั่วไปจะนานกว่านั้นสักหลายวัน

### ลำดับการเพิ่มการ์ด (Insertion Order)

ควบคุมว่า Anki ควรเพิ่มการ์ดใหม่ในสำรับแบบสุ่มหรือแบบเรียงลำดับ  
เมื่อคุณเปลี่ยนตัวเลือกนี้ Anki จะจัดเรียงสำรับในค่าปริยายปัจจุบันใหม่

ในเวอร์ชัน Anki ล่าสุด ควรตั้งค่านี้เป็น `Sequential` และปรับ [ลำดับการแสดงผล](deck-options.md#display-order) แทน

## การล้มเหลว (Lapses)

เมื่อคุณคลิก **Again** บนการ์ดทบทวน จะเรียกว่า _lapse_ ตัวเลือกในส่วนนี้จะส่งผลกับการ์ดที่ล้มเหลวเช่นนี้

### ขั้นตอนการเรียนรู้ซ้ำ (Relearning Steps)

เหมือนกับขั้นตอนการเรียนรู้ แต่ใช้กับการ์ดที่ล้มเหลว  
เมื่อคุณล้มเหลวในการทบทวนการ์ด (กด **Again**) การ์ดจะผ่าน _ขั้นตอนการเรียนรู้ซ้ำ_ ก่อนที่จะกลายเป็นการ์ดทบทวนอีกครั้ง

ถ้าคุณเว้นว่างขั้นตอนนี้ การ์ดจะข้ามการเรียนรู้ซ้ำ และจะถูกตั้งช่วงเวลาหน่วงใหม่เป็น 1 วันตามค่าเริ่มต้น

### ระยะเวลาขั้นต่ำ (Minimum Interval)

กำหนดจำนวนวันที่น้อยที่สุดที่การ์ดควรรอหลังจากจบการเรียนรู้ซ้ำ  
ค่าเริ่มต้นคือ 1 วัน หมายความว่าหลังจากเรียนรู้ซ้ำเสร็จ การ์ดจะแสดงอีกครั้งในวันถัดไป

### การ์ดติด (Leeches)

ควบคุมวิธีที่ Anki จัดการกับการ์ดติด รายละเอียดดูได้ในส่วน [leeches](leeches.md)

## ลำดับการแสดงผล (Display Order)

ตัวเลือกในส่วนนี้มาจากสำรับที่คุณเลือกเรียน ไม่ใช่สำรับของการ์ดที่กำลังแสดงอยู่

ข้อมูลเพิ่มเติมเกี่ยวกับลำดับการแสดงผลดูได้ใน [ส่วนการเรียน](studying.md#display-order)

### ลำดับการเก็บการ์ดใหม่ (New Card Gather Order)

ควบคุมวิธีที่ Anki ดึงการ์ดใหม่จากสำรับ ตัวเลือกมีดังนี้:

- **สำรับ (Deck)**: ดึงการ์ดจากสำรับย่อยแต่ละอันตามลำดับ เริ่มจากสำรับย่อยบนสุด  
  การ์ดจากสำรับย่อยแต่ละอันจะถูกดึงตามลำดับตำแหน่งเพิ่มขึ้น  
  ถ้าถึงขีดจำกัดรายวันของสำรับที่เลือกเรียนแล้ว อาจหยุดดึงก่อนจะตรวจสอบสำรับย่อยทั้งหมด  
  วิธีนี้เร็วที่สุดในชุดสะสมขนาดใหญ่ และช่วยให้คุณตั้งลำดับความสำคัญกับสำรับย่อยที่อยู่ด้านบน

  สำรับ/สำรับย่อยจะเรียงลำดับตามตัวอักษรเสมอ คุณสามารถตั้งชื่อโดยใส่เลขนำหน้าเช่น 001 เพื่อควบคุมลำดับการแสดง  
  คุณยังสามารถใช้ `_` และ `~` เป็นเลขนำหน้าเพื่อจัดวางให้อยู่บนสุดหรือล่างสุดได้

  แม้ว่าลำดับตำแหน่งจะขึ้นกับตัวเลือก insertion order ในตอนแรก คุณสามารถ [ปรับตำแหน่ง](./browsing.md#cards) การ์ดด้วยวิธีต่าง ๆ ได้เอง

- **สำรับ แล้วโน้ตสุ่ม (Deck, then random notes)**: ดึงการ์ดจากสำรับย่อยแต่ละอันตามลำดับ เริ่มจากบนสุด  
  การ์ดจากแต่ละสำรับย่อยจะถูกดึงจากโน้ตที่เลือกแบบสุ่ม

- **ตำแหน่งเพิ่มขึ้น (Ascending position)**: ดึงการ์ดโดยเรียงตามตำแหน่งเพิ่มขึ้น (due #) ซึ่งมักเป็นการ์ดที่เพิ่มเข้ามานานที่สุดก่อน

- **ตำแหน่งลดลง (Descending position)**: ดึงการ์ดโดยเรียงตามตำแหน่งลดลง (due #) ซึ่งมักเป็นการ์ดที่เพิ่มเข้ามาใหม่ล่าสุดก่อน

- **โน้ตสุ่ม (Random notes)**: ดึงการ์ดจากโน้ตที่เลือกแบบสุ่ม

- **การ์ดสุ่ม (Random cards)**: ดึงการ์ดในลำดับสุ่ม

### ลำดับการจัดเรียงการ์ดใหม่ (New Card Sort Order)

ควบคุมวิธีการจัดเรียงการ์ดใหม่หลังจากที่ดึงมาแล้ว ตัวเลือกมีดังนี้:

- **ประเภทการ์ด แล้วตามลำดับที่ดึงมา**: แสดงการ์ดตามหมายเลขประเภทการ์ด โดยการ์ดแต่ละประเภทจะแสดงตามลำดับที่ถูกดึงมา  
  ถ้าปิดการซ่อนการ์ดพี่น้อง (sibling burying) จะทำให้การ์ดแบบหน้า→หลังทั้งหมดแสดงก่อนการ์ดแบบหลัง→หน้า  
  วิธีนี้เหมาะถ้าคุณไม่ต้องการให้การ์ดพี่น้องแสดงชิดกันเกินไป

- **ลำดับที่ดึงมา**: แสดงการ์ดตามลำดับที่ดึงมาเป๊ะ ๆ  
  ถ้าปิดการซ่อนการ์ดพี่น้อง โดยปกติจะเห็นการ์ดพี่น้องทั้งหมดแสดงต่อกันไป

- **ประเภทการ์ด แล้วสุ่ม**: แสดงการ์ดตามหมายเลขประเภทการ์ด แต่สลับการ์ดในแต่ละประเภทแบบสุ่ม  
  วิธีนี้เหมาะถ้าคุณไม่ต้องการให้การ์ดพี่น้องแสดงชิดกันเกินไป แต่ยังอยากให้การ์ดแสดงแบบสุ่ม

- **โน้ตสุ่ม แล้วประเภทการ์ด**: เลือกโน้ตแบบสุ่ม แล้วแสดงการ์ดพี่น้องทั้งหมดตามลำดับ

- **สุ่ม**: สลับการ์ดทั้งหมดแบบสุ่มเต็มที่

### ลำดับการ์ดใหม่/ทบทวน (New/Review Order)

ควรกำหนดว่าการ์ดใหม่จะผสมกับการ์ดทบทวน แสดงก่อน หรือแสดงหลัง

### ลำดับการเรียนรู้/ทบทวนข้ามวัน (Interday Learning/Review Order)

ควรกำหนดว่าการ์ด (เรียน) ที่ข้ามขอบเขตวัน จะผสมกับการ์ดทบทวน แสดงก่อน หรือแสดงหลัง  
เนื่องจากการ์ดเรียนรู้มักยากกว่าการ์ดทบทวน บางคนจึงชอบเห็นการ์ดเหล่านี้ตอนท้าย (ทำของง่ายก่อน) หรือเห็นตอนต้น (ให้เวลาทบทวนการ์ดลืมมากขึ้น)

### ลำดับการจัดเรียงการ์ดทบทวน (Review Sort Order)

ควบคุมวิธีการจัดเรียงการ์ดทบทวน ตัวเลือกมีดังนี้:

- **วันครบกำหนด แล้วสุ่ม**: ลำดับเริ่มต้นจะให้ความสำคัญกับการ์ดที่รอนานกว่า  
  แนะนำให้ใช้เมื่อตามทบทวนทัน หรือมีการ์ดค้างน้อย  
  ถ้าหยุดพักนานหรือตามทบทวนไม่ทัน อาจเปลี่ยนลำดับชั่วคราวได้

- **วันครบกำหนด แล้วสำรับ**: ให้ความสำคัญกับการ์ดที่รอนานกว่า แล้วแสดงการ์ดทบทวนของแต่ละสำรับย่อยตามลำดับ

- **สำรับ แล้ววันครบกำหนด**: แสดงการ์ดทบทวนของแต่ละสำรับย่อยตามลำดับ  
  โดยทั่วไปไม่แนะนำเพราะทำให้การ์ดแสดงในลำดับเดิมซ้ำ ๆ ง่ายต่อการทายคำตอบจากบริบท และทำให้ความจำอ่อนแอ

- **ช่วงเวลาสั้นขึ้น (Ascending intervals)**: แสดงการ์ดที่มีช่วงเวลาระหว่างทบทวนสั้นกว่าก่อน

- **ช่วงเวลานานขึ้น (Descending intervals)**: แสดงการ์ดที่มีช่วงเวลาระหว่างทบทวนยาวกว่าก่อน

- **ความง่ายน้อยขึ้น (Ascending ease)**: แสดงการ์ดที่ยากกว่าก่อน

- **ความง่ายมากขึ้น (Descending ease)**: แสดงการ์ดที่ง่ายกว่าก่อน

- **ความเก่าที่สัมพันธ์ (Relative overdueness)**: แสดงการ์ดที่คุณน่าจะลืมมากกว่าก่อน  
  โดยทั่วไปแนะนำถ้าคุณมีการ์ดค้างเยอะและต้องการลดโอกาสลืมการ์ดมากขึ้น

  เมื่อใช้ алгоритм SM-2, ความเก่าจะถูกประเมินโดยเทียบระหว่างความเก่าของการ์ดและช่วงเวลาทบทวน  
  เช่น การ์ดที่มีช่วงเวลาปัจจุบัน 5 วัน และเกินกำหนด 2 วัน จะแสดงก่อนการ์ดที่มีช่วงเวลาปัจจุบัน 10 วัน และเกินกำหนด 3 วัน

  เมื่อเปิดใช้ FSRS ลำดับการจัดเรียงนี้จะถูกลบออก; ลำดับเทียบเท่าคือ **Ascending retrievability**  
  ซึ่งคำนวณจากความสามารถในการจำของแต่ละการ์ดและการเก็บรักษาที่ตั้งไว้ในค่าปริยาย

## การซ่อนการ์ด (Burying)

เมื่อ Anki ดึงการ์ด จะดึงการ์ดเรียนรู้ในวันนั้นก่อน, ตามด้วยการ์ดเรียนรู้ข้ามวัน, การ์ดทบทวน, และสุดท้ายการ์ดใหม่  
สิ่งนี้ส่งผลต่อการทำงานของการซ่อนการ์ดดังนี้:

- ถ้าคุณเปิดใช้งานตัวเลือกการซ่อนการ์ดทั้งหมด การ์ดพี่น้องที่อยู่ก่อนในลำดับนี้จะถูกแสดง เช่น การ์ดทบทวนจะแสดงก่อนการ์ดใหม่
- การ์ดพี่น้องที่อยู่หลังในลำดับนี้ไม่สามารถซ่อนการ์ดประเภทก่อนหน้าได้  
  เช่น ถ้าปิดการซ่อนการ์ดใหม่ แล้วเรียนการ์ดใหม่ จะไม่ซ่อนการ์ดเรียนรู้ข้ามวันหรือการ์ดทบทวน และคุณอาจเห็นทั้งการ์ดทบทวนและการ์ดใหม่ในเซสชันเดียวกัน

ตัวเลือกมีดังนี้:

- **ซ่อนการ์ดพี่น้องใหม่**: ว่าการ์ดใหม่อื่น ๆ ของโน้ตเดียวกัน (เช่น การ์ดกลับด้าน หรือช่องลบที่ติดกัน) จะถูกหน่วงจนถึงวันถัดไปหรือไม่

- **ซ่อนการ์ดพี่น้องทบทวน**: ว่าการ์ดทบทวนอื่น ๆ ของโน้ตเดียวกันจะถูกหน่วงจนถึงวันถัดไปหรือไม่

- **ซ่อนการ์ดพี่น้องเรียนรู้ข้ามวัน**: ว่าการ์ดเรียนรู้ข้ามวันที่เหลือของโน้ตเดียวกันจะถูกหน่วงจนถึงวันถัดไปหรือไม่

สำหรับข้อมูลเพิ่มเติมเกี่ยวกับการซ่อนการ์ด ดู [ส่วนนี้](./studying.md#siblings-and-burying) ในคู่มือ

## เสียง (Audio)

- **ไม่เล่นเสียงอัตโนมัติ**: โดยปกติ Anki จะเล่นเสียงอัตโนมัติจากการ์ด  
  ถ้าเปิดตัวเลือกนี้ Anki จะไม่เล่นเสียงจนกว่าคุณจะกดปุ่มเล่นเสียงซ้ำ <kbd>R</kbd> หรือ <kbd>F5</kbd>

- **ข้ามเสียงคำถามเมื่อเล่นเสียงคำตอบซ้ำ**: ควบคุมว่าเสียงจากด้านคำถามจะเล่นเมื่อใช้การเล่นซ้ำเสียงที่ด้านคำตอบหรือไม่  
  โปรดทราบว่า Anki [จะไม่เล่นเสียงอัตโนมัติ](./templates/fields.md#special-fields) จากช่อง `{{FrontSide}}` ตัวเลือกนี้ไม่ส่งผลต่อพฤติกรรมการเล่นเสียงอัตโนมัติ

## ตัวจับเวลา (Timers)
<a id="timer"></a>

Anki ตรวจจับเวลาที่คุณใช้ตอบแต่ละการ์ด เพื่อแสดงเวลาที่ใช้เรียนในแต่ละวัน  
เวลาในการตอบไม่มีผลต่อการจัดตาราง

### ตัวจับเวลาในโปรแกรม (Internal Timer)

- วินาทีตอบสูงสุด: ค่าปริยายคือ 60 วินาที  
  หากใช้เวลานานกว่านั้น Anki จะถือว่าคุณเดินออกจากคอมพิวเตอร์หรือถูกเบี่ยงเบนความสนใจ  
  และจำกัดเวลาที่บันทึกไว้ไม่เกิน 60 วินาที เพื่อป้องกันสถิติไม่ถูกต้อง  
- ตัวจับเวลานี้เริ่มทำงานตั้งแต่แสดงคำถามจนกว่าคุณจะกดปุ่มให้คะแนนคำตอบ  
  ถ้าคุณใช้เวลานานกว่า 60 วินาทีเป็นประจำ  
  คุณอาจพิจารณาปรับเพิ่มขีดจำกัดนี้ หรือถ้าเป็นไปได้ ควรทำการ์ดให้ง่ายขึ้น

### ตัวจับเวลาบนหน้าจอ (On-screen Timer)

- แสดงตัวจับเวลาบนหน้าจอ: ในหน้าจอเรียน จะแสดงตัวจับเวลาที่นับเวลาที่คุณใช้เรียนการ์ดแต่ละใบ (ตัวจับเวลานี้จะหยุดเมื่อถึง วินาทีตอบสูงสุด ที่ตั้งไว้ในตัวจับเวลาในโปรแกรม)  
- หยุดตัวจับเวลาบนหน้าจอเมื่อแสดงคำตอบ: กำหนดว่าตัวจับเวลาบนหน้าจอจะยังนับต่อหลังจากแสดงคำตอบจนกว่าจะกดปุ่มให้คะแนนคำตอบหรือไม่  
  ตัวเลือกนี้ไม่มีผลกับเวลาที่บันทึกเพื่อสถิติของคุณ

## เลื่อนหน้าอัตโนมัติ (Auto Advance)

ต้องใช้ Anki เวอร์ชัน 23.12 ขึ้นไป เลื่อนหน้าอัตโนมัติช่วยให้คุณทำบางอย่างโดยอัตโนมัติหลังจากเวลาที่กำหนดผ่านไป  
ก่อนใช้งาน ต้องตั้งเวลาใน **วินาทีที่แสดงคำถาม** และ/หรือ **วินาทีที่แสดงคำตอบ** ให้ไม่เป็นศูนย์ก่อน  
จากนั้นในหน้าจอเรียน ใช้คำสั่งเลื่อนหน้าอัตโนมัติจากปุ่ม **More** เพื่อเริ่มเลื่อนหน้า

## วันง่าย (Easy Days)

ถ้าคุณต้องการใช้เวลาน้อยลงกับ Anki บางวันในสัปดาห์ เช่น วันอาทิตย์ ฟีเจอร์นี้ช่วยได้  
หลังจากคำนวณช่วงเวลาเสร็จ จะปรับเล็กน้อยเพื่อเปลี่ยนวันครบกำหนด  
โปรดทราบว่าการตั้งวันทั้งหมดเป็น "ลดลง" หรือ "ขั้นต่ำ" จะทำให้ภาระงานเท่ากับตั้งวันทั้งหมดเป็น "ปกติ"  
ฟีเจอร์นี้ใช้งานได้กับทั้ง FSRS และอัลกอริทึม SM-2 แบบเดิม  
การเปลี่ยนการตั้งค่า Easy Days จะไม่เปลี่ยนช่วงเวลาที่มีอยู่ย้อนหลัง และจะส่งผลเฉพาะช่วงเวลาในอนาคต  
พูดง่าย ๆ คือ คุณจะไม่เห็นการเปลี่ยนแปลงทันทีในจำนวนการ์ดครบกำหนด

## FSRS

[Free Spaced Repetition Scheduler (FSRS)](https://github.com/open-spaced-repetition/fsrs4anki/wiki/ABC-of-FSRS) เป็นทางเลือกแทนอัลกอริทึม SuperMemo 2 (SM-2) แบบเดิมของ Anki  
ด้วยการคำนวณว่าคุณน่าจะลืมข้อมูลมากแค่ไหนอย่างแม่นยำขึ้น  
FSRS ช่วยให้คุณจำเนื้อหาได้มากขึ้นในเวลาที่เท่ากัน

เมื่อเปิดใช้ FSRS จะมีตัวเลือกใหม่ปรากฏขึ้น  
และตัวเลือกเฉพาะ SM-2 เช่น **ช่วงเวลาหลังจบการเรียนรู้**  
**โบนัสง่าย** ฯลฯ จะถูกซ่อน ตัวเลือกนี้ใช้กับทุก preset

**ก่อนเปิดใช้**

- โปรดตรวจสอบว่าไคลเอนต์ Anki ทั้งหมดของคุณรองรับ FSRS  
  Anki 23.10, AnkiMobile 23.10 และ AnkiWeb รองรับทั้งหมด  
  AnkiDroid รองรับในเวอร์ชัน 2.17+  
  หากไคลเอนต์ใดไม่รองรับ จะใช้งานไม่ถูกต้อง
- หากเคยใช้เวอร์ชัน "custom scheduling" ของ FSRS กรุณาล้างส่วน custom scheduling ก่อนเปิดใช้ FSRS

### คู่มือสั้น ๆ

- เปิด FSRS ในส่วน "FSRS" ที่ท้ายหน้าตัวเลือกสำรับ FSRS เปิดได้เฉพาะแบบทั่วโลก ไม่สามารถเปิดแค่บาง preset และปิดบาง preset ได้
- ตรวจสอบให้แน่ใจว่าขั้นตอนเรียนรู้และเรียนรู้ซ้ำทั้งหมดสั้นกว่า 1 วัน และสามารถทำให้เสร็จภายในวันเดียวกัน  
  ไม่แนะนำให้ใช้ 23 ชั่วโมง แม้จะน้อยกว่า 1 วัน เพราะจะทำให้ไม่สามารถเรียนจบขั้นตอนในวันเดียวกับการทบทวนครั้งแรก  
  ขั้นตอนอย่าง 10 นาที หรือ 30 นาที เหมาะสมกว่า
- คลิกปุ่ม "Optimize" ในช่อง "FSRS parameters" หากเห็นข้อความว่า "The FSRS parameters currently appear to be optimal" ก็ถือว่าโอเค
- เลือกค่าคงทนที่ต้องการ (desired retention): สัดส่วนการ์ดที่จำได้เมื่อครบกำหนด  
  **นี่คือการตั้งค่าที่สำคัญที่สุดของ FSRS ยิ่งค่าคงทนสูง ช่วงเวลาจะสั้นลงและต้องทบทวนมากขึ้นต่อวัน**  
  ค่าเริ่มต้นคือ 90% ซึ่งให้สมดุลที่ดีระหว่างการจำและภาระงาน  
  เกิน 90% ภาระงานจะเพิ่มขึ้นอย่างรวดเร็ว  
  เกิน 97% อาจหนักเกินไป  
  คุณสามารถใช้ ["คำนวณค่าคงทนขั้นต่ำที่แนะนำ"](#compute-minimum-recommended-retention) เพื่อช่วยเลือกค่าได้  
  พารามิเตอร์และค่าคงทนเป็นของแต่ละ preset คุณสามารถสร้าง preset หลายอันด้วยพารามิเตอร์และค่าคงทนต่างกันได้

FSRS ปรับตัวเข้ากับนิสัยเกือบทุกอย่าง ยกเว้นอย่างเดียวคือ การกด "Hard" แทน "Again" เมื่อคุณลืมข้อมูล  
เมื่อกด "Hard" FSRS จะถือว่าคุณจำข้อมูลได้ถูกต้อง (แม้จะลังเลและใช้ความพยายามมาก)  
ถ้าคุณกด "Hard" ในขณะที่ลืมข้อมูล ช่วงเวลาทุกช่วงจะสูงเกินจริง  
ดังนั้น ถ้าคุณมีนิสัยนี้ โปรดเปลี่ยนและใช้ "Again" เมื่อคุณลืมข้อมูล

เกี่ยวกับความเข้ากันได้กับ add-on โดยทั่วไป หาก add-on มีผลต่อช่วงเวลาและการจัดตารางในทางใดทางหนึ่ง ไม่ควรใช้ร่วมกับ FSRS

### การคงทนที่ต้องการ (Desired Retention)

การคงทนที่ต้องการควบคุมความน่าจะเป็นที่คุณจะจำการ์ดได้เมื่อถึงเวลาทบทวน  
ค่าปริยาย `0.90` จะตั้งเวลาทบทวนการ์ดให้คุณมีโอกาสจำได้ 90% เมื่อทบทวนอีกครั้ง  
โดยทั่วไปหมายความว่าคุณจะจำได้ประมาณ 90% ของการ์ดที่ทบทวน และล้มเหลวประมาณ 10% เท่านั้น

นี่คือกราฟที่แสดงว่าเมื่อปรับค่านี้ จะส่งผลต่อภาระงานของคุณอย่างไร:

![graph showing an exponential increase in workload as desired retention nears one.](./media/FSRS_retention.png)

รูปร่างของกราฟจะแตกต่างกันในแต่ละคน แต่มีรูปแบบหลักสองอย่างที่เป็นจริงสำหรับทุกคน:

- เมื่อการคงทนที่ต้องการเข้าใกล้ 1.0 ภาระงานจะเพิ่มขึ้นอย่างมาก  
  สมมติว่าคุณมีการ์ดที่มีโอกาสจำได้ 90% หลัง 100 วัน  
  ถ้าคุณตั้งค่าคงทนที่ต้องการเป็น `0.90` คุณจะทบทวนการ์ดนั้นอีกครั้งหลัง 100 วัน  
  แต่ถ้าค่าคงทนที่ต้องการเป็น `0.95` คุณจะต้องทบทวนหลัง 46 วันแทน  
  หมายความว่าช่วงเวลาของการ์ดจะลดลงเกือบครึ่งเมื่ออยู่ที่ `0.95` และคุณต้องทบทวนบ่อยขึ้นสองเท่าจาก `0.90`  
  ที่ `0.97` ช่วงเวลาจะเหลือ 27 วัน (ต้องทบทวนบ่อยขึ้น 3.7 เท่า)  
  ที่ `0.99` ช่วงเวลาจะเหลือเพียง 9 วัน (ต้องทบทวนบ่อยขึ้นกว่า 10 เท่าจากค่าปริยาย)

- เมื่อการคงทนที่ต้องการลดลง คุณจะลืมการ์ดจำนวนมากขึ้น และต้องทบทวนการ์ดเหล่านั้นซ้ำ  
  จนในที่สุด การ์ดที่ลืมจะทำให้ภาระงานเพิ่มขึ้นมากกว่าประโยชน์จากการเลื่อนวันครบกำหนดที่ยาวขึ้น  
  นอกจากนี้ การลืมเนื้อหาบ่อยๆ ยังทำให้หมดกำลังใจได้

ด้วยเหตุผลเหล่านี้ เราแนะนำให้คุณปรับค่านี้อย่างระมัดระวัง และแนะนำให้ตั้งต่ำกว่า `0.97` แต่สูงกว่าค่า [การคงทนขั้นต่ำที่แนะนำ](#compute-minimum-recommended-retention)

### พารามิเตอร์ FSRS (FSRS Parameters)

พารามิเตอร์ FSRS มีผลต่อการตั้งเวลาทบทวนการ์ด อย่าปรับแต่งพารามิเตอร์เหล่านี้ด้วยตนเอง หรือคัดลอกจากผู้อื่น

**ปรับแต่งพารามิเตอร์ FSRS**

เครื่องมือปรับแต่ง FSRS ใช้การเรียนรู้ของเครื่องเพื่อวิเคราะห์รูปแบบความจำของคุณ และหาพารามิเตอร์ที่เหมาะสมกับประวัติการทบทวนของคุณ  
โดยจะต้องมีการทบทวนหลายครั้งเพื่อปรับแต่งพารามิเตอร์ให้เหมาะสม

เมื่อคุณคลิกปุ่ม **Optimize** FSRS จะวิเคราะห์ประวัติการทบทวนและสร้างพารามิเตอร์ที่เหมาะสมกับความจำและเนื้อหาที่คุณเรียน  
ถ้าคุณมีสำรับที่ความยากแตกต่างกันมาก แนะนำให้ตั้ง preset แยกกัน เพราะพารามิเตอร์ของสำรับง่ายจะต่างจากสำรับยาก  
ไม่จำเป็นต้องปรับแต่งพารามิเตอร์บ่อย ๆ เดือนละครั้งก็เพียงพอ

โดยค่าปริยาย พารามิเตอร์จะถูกคำนวณจากประวัติการทบทวนของทุกสำรับที่ใช้ preset ปัจจุบัน  
คุณสามารถ [ปรับการค้นหา](./searching.md) ก่อนปรับแต่งพารามิเตอร์ได้ หากต้องการเปลี่ยนการ์ดที่ใช้ในการปรับแต่ง

คุณยังสามารถปรับแต่งพารามิเตอร์สำหรับทุก preset ของคุณพร้อมกันได้โดยคลิก **Optimize All Presets**

**ประเมินพารามิเตอร์ FSRS**

คุณสามารถใช้ปุ่ม **Evaluate** เพื่อดูเมตริกที่แสดงว่าพารามิเตอร์เหมาะสมกับประวัติการทบทวนของคุณแค่ไหน  
ค่ายิ่งเล็กแสดงว่าพารามิเตอร์เหมาะสมมากขึ้น

Log loss ไม่มีความหมายที่เข้าใจง่าย ส่วน RMSE (bins) แสดงค่าความแตกต่างเฉลี่ยระหว่างความน่าจะเป็นที่คาดการณ์ว่าจะจำได้ (R) กับความน่าจะเป็นจริงจากประวัติการทบทวน  
เช่น RMSE=5% หมายความว่าโดยเฉลี่ย FSRS คาดการณ์ค่า R ผิดพลาดประมาณ 5%  
คุณไม่จำเป็นต้องเข้าใจเมตริกเหล่านี้เพื่อใช้ FSRS

โปรดทราบว่า log loss กับ RMSE (bins) อาจไม่สัมพันธ์กันเสมอไป  
สำรับสองชุดอาจมีค่า RMSE ใกล้เคียงกัน แต่ log loss ต่างกันมาก หรือในทางกลับกัน

โดยค่าปริยาย log loss และ RMSE (bins) คำนวณจากทุกสำรับที่ใช้ preset ปัจจุบัน  
คุณสามารถ [ปรับการค้นหา](./searching.md) ก่อนประเมินพารามิเตอร์ได้ หากต้องการเปลี่ยนการ์ดที่ใช้ในการประเมิน

### ตั้งเวลาทบทวนใหม่เมื่อเปลี่ยนแปลง (Reschedule Cards on Change)

ตัวเลือกนี้ควบคุมว่าจะเปลี่ยนวันครบกำหนดของการ์ดหรือไม่ เมื่อเปิดใช้ FSRS เปลี่ยนค่าคงทนที่ต้องการ หรือเปลี่ยนพารามิเตอร์  
ค่าปริยายคือไม่ตั้งเวลาทบทวนใหม่ การทบทวนในอนาคตจะใช้การตั้งเวลาแบบใหม่ แต่ภาระงานจะไม่เปลี่ยนทันที  
หากเปิดใช้ การ์ดจะถูกตั้งเวลาทบทวนใหม่ ซึ่งอาจทำให้จำนวนการ์ดครบกำหนดเพิ่มมาก  
**จึงไม่แนะนำให้เปิดใช้ตัวเลือกนี้เมื่อเพิ่งเปลี่ยนจาก SM-2**

ใช้ตัวเลือกนี้อย่างระมัดระวัง เพราะจะเพิ่มบันทึกการทบทวนในแต่ละการ์ด และเพิ่มขนาดฐานข้อมูลของคุณ

ถ้าคุณเพิ่งเปลี่ยนจาก SM-2 และยังต้องการใช้ตัวเลือกนี้ แนะนำให้สำรองข้อมูลก่อน เปิดใช้ FSRS พร้อมตั้งเวลาทบทวนใหม่ และถ้าจำเป็น คุณสามารถยกเลิกหรือกู้คืนจากข้อมูลสำรองได้

### คำนวณการคงทนขั้นต่ำที่แนะนำ (Compute Minimum Recommended Retention)

การคำนวณการคงทนขั้นต่ำที่แนะนำ (CMRR) พยายามหาค่าการคงทนที่ต้องการซึ่งทำให้เรียนรู้เนื้อหาได้มากที่สุดในเวลาที่น้อยที่สุด  
ค่าที่ได้สามารถใช้เป็นเกณฑ์อ้างอิงในการตั้งค่าการคงทนที่ต้องการ  
คุณอาจเลือกค่าการคงทนที่สูงขึ้น ถ้าพร้อมแลกกับเวลาศึกษาที่มากขึ้นเพื่อความจำที่ดีกว่า  
แต่ไม่แนะนำให้ตั้งค่าการคงทนที่ต่ำกว่าค่าขั้นต่ำนี้ เพราะจะทำให้ต้องใช้เวลาศึกษามากเกินความจำเป็นเนื่องจากลืมบ่อยขึ้น

### ตัวจำลอง (The Simulator)

คุณสามารถใช้ตัวจำลองเพื่อประเมินภาระงานของคุณ  
ไม่ว่าจะเป็นจำนวนการทบทวนต่อวัน หรือจำนวนเวลาศึกษาต่อวัน (นาที)

- **จำนวนวันจำลอง** ควบคุมระยะเวลาของประวัติการศึกษาที่จำลองขึ้น  
- **จำนวนการ์ดใหม่เพิ่มเติมที่จำลอง** ควบคุมว่าตัวจำลองจะจำลองการ์ดเพิ่มกว่าที่มีใน preset นี้หรือไม่  
  เช่น หากคุณมีการ์ด 100 ใบใน preset นี้ และตั้งค่านี้เป็น 50 ตัวจำลองจะจำลองทั้งหมด 150 ใบ  
  ซึ่งจะมีประโยชน์ถ้าคุณวางแผนสร้างการ์ดใหม่มากขึ้นในอนาคต  
- **การ์ดใหม่/วัน** และ **จำนวนทบทวนสูงสุด/วัน** ควบคุมจำนวนการ์ดใหม่ที่เรียนรู้ในแต่ละวัน และจำนวนทบทวนสูงสุดต่อวัน  
- **ช่วงเวลาสูงสุด** ควบคุมความยาวช่วงเวลาสูงสุด (เป็นวัน)

เพื่อให้การจำลองสมจริงที่สุด ตัวจำลองจะพิจารณาสถานะความจำจริง (ความยาก, ความมั่นคง, ความสามารถในการเรียกคืน) ของการ์ดของคุณ  
นอกจากนี้ยังใช้พารามิเตอร์ FSRS และค่าการคงทนที่ต้องการด้วย ดังนั้นการเปลี่ยนแปลงค่าเหล่านี้จะส่งผลต่อการจำลอง

#### ขั้นตอนการเรียนรู้และการเรียนรู้ซ้ำ

ไม่แนะนำให้ใช้ขั้นตอน (re)learning ที่มีระยะเวลา 1 วันขึ้นไปเมื่อใช้ FSRS  
เหตุผลหลักที่ขั้นตอนเหล่านี้นิยมกับอัลกอริทึม SM-2 เดิมคือ  
การล้มเหลวซ้ำๆ หลังจากการ์ดผ่านขั้นตอนเรียนรู้แล้ว อาจลดความง่ายของการ์ดอย่างมาก จนเกิดสิ่งที่บางคนเรียกว่า “นรกความง่าย”  
ซึ่ง FSRS ไม่มีปัญหานี้  
การตั้งขั้นตอนการเรียนรู้ให้ต่ำกว่าหนึ่งวัน จะช่วยให้ FSRS ตั้งเวลาได้เหมาะสมกับเนื้อหาและความจำของคุณมากกว่า  
เหตุผลอีกอย่างที่ไม่ควรใช้ขั้นตอนเรียนรู้ที่ยาวคือ FSRS อาจตั้งเวลาทบทวนครั้งแรกสั้นกว่าขั้นตอนเรียนรู้ล่าสุดของคุณ ทำให้ปุ่ม **Hard** แสดงเวลานานกว่าปุ่ม **Good**

เรายังแนะนำให้ลดจำนวนขั้นตอนเรียนรู้ให้น้อยที่สุด  
มีงานวิจัยแสดงว่าการทบทวนการ์ดหลายครั้งในวันเดียวกันไม่ได้ช่วยเพิ่มความจำระยะยาวอย่างมีนัยสำคัญ  
เวลาของคุณจะดีกว่าถ้าใช้กับการ์ดอื่น ๆ หรือลดเวลาการศึกษาต่อครั้ง

ในเวอร์ชันล่าสุดของ Anki คุณสามารถปล่อยช่องขั้นตอน (re)learning ว่างไว้ เพื่อให้ FSRS ควบคุมการตั้งเวลาระยะสั้นได้  
นี่เป็นฟีเจอร์ทดลอง  
โปรดทราบว่า FSRS-5 *สามารถ* ให้ช่วงเวลาสั้นกว่าหนึ่งวันได้ ไม่ได้หมายความว่า *จะ* ให้เสมอ  
ช่วงเวลา **Again** อาจเป็นหนึ่งวัน หรือมากกว่านั้นได้

## ขั้นสูง

### ช่วงเวลาสูงสุด (Maximum Interval)

จำนวนวันสูงสุดที่การ์ดทบทวนจะรอจนกว่าจะถูกแสดงอีกครั้ง  
เมื่อถึงขีดจำกัดนี้ ปุ่ม **Hard**, **Good** และ **Easy** จะให้ช่วงเวลาเดียวกัน  
ถ้าตั้งค่านี้สั้นลง ภาระงานของคุณจะมากขึ้น  
ค่าปริยายคือ 100 ปี คุณสามารถลดลงเพื่อแลกกับเวลาศึกษาที่มากขึ้นเพื่อความคงทนที่สูงกว่า

### การคงทนในอดีต (Historical Retention)

การตั้งค่านี้ซ่อนอยู่จนกว่าจะเปิดใช้ FSRS

เมื่อประวัติการทบทวนของคุณขาดหายไป FSRS จำเป็นต้องเติมช่องว่างนั้น  
ค่าปริยายคือสมมติว่าคุณจำได้ 90% ในช่วงเวลาที่ขาดหายไปนั้น  
ถ้าการคงทนในอดีตของคุณสูงหรือต่ำกว่ามากกว่า 90% การปรับค่านี้จะช่วยให้ FSRS ประมาณผลได้แม่นยำขึ้น

ประวัติการทบทวนของคุณอาจไม่สมบูรณ์ด้วยสองเหตุผล:

- เพราะคุณใช้ตัวเลือก **Ignore cards reviewed before**  
- เพราะคุณลบบันทึกการทบทวนเก่าเพื่อเพิ่มพื้นที่ หรือย้ายข้อมูลมาจากโปรแกรม SRS อื่น

กรณีหลังพบได้ไม่บ่อย ดังนั้นถ้าไม่ได้ใช้ตัวเลือกแรก คุณอาจไม่จำเป็นต้องปรับค่านี้

### ข้ามการ์ดที่ทบทวนก่อนหน้า

ถ้าตั้งค่าไว้ การ์ดที่ทบทวนก่อนวันที่ที่กำหนดจะถูกข้ามเมื่อทำการปรับแต่งพารามิเตอร์ FSRS  
ซึ่งมีประโยชน์ถ้าคุณนำเข้าข้อมูลการตั้งเวลาของคนอื่น หรือเปลี่ยนวิธีการใช้ปุ่มตอบคำถาม

### ค่าเริ่มต้นความง่าย (Starting Ease)

ควบคุมค่าความง่ายที่การ์ดเริ่มต้นด้วย  
ตั้งค่าเมื่อตัวการ์ดจบขั้นตอนการเรียนรู้ครั้งแรก  
ค่าปริยายคือ 2.50 หมายความว่าเมื่อเรียนจบแล้ว การตอบ **Good** ในการทบทวนครั้งต่อ ๆ ไป จะเพิ่มช่วงเวลาทบทวนโดยประมาณ 2.5 เท่า  
(เช่น ถ้าช่วงเวลาก่อนหน้าเป็น 10 วัน ช่วงเวลาต่อไปจะประมาณ 25 วัน)  
ค่าความง่ายอาจเพิ่มหรือลดได้ขึ้นกับการให้คะแนนในการทบทวนครั้งถัดไป

### โบนัสสำหรับปุ่ม Easy (Easy Bonus)

เป็นตัวคูณเพิ่มที่นำไปใช้กับช่วงเวลาทบทวนเมื่อการ์ดทบทวนถูกตอบด้วย **Easy**  
ค่าปริยายคือ 1.30 หมายความว่า ช่วงเวลาของปุ่ม Easy จะเป็น 1.3 เท่าของช่วงเวลาของปุ่ม Good  
(เช่น ถ้าช่วงเวลาปุ่ม Good คือ 10 วัน ช่วงเวลาปุ่ม Easy จะประมาณ 13 วัน)

### ตัวคูณช่วงเวลา (Interval Modifier)

ตัวคูณเพิ่มที่นำไปใช้กับทุกการทบทวน  
ค่าปริยายคือ 1.00 หมายความว่าไม่มีการเปลี่ยนแปลง  
ถ้าตั้งเป็น 0.80 ช่วงเวลาจะถูกลดเหลือ 80% ของช่วงเวลาปกติ  
(เช่น ช่วงเวลา 10 วัน จะกลายเป็น 8 วัน)  
คุณสามารถใช้ตัวคูณนี้เพื่อทำให้การทบทวนถี่ขึ้นหรือน้อยลงได้

สำหรับเนื้อหาที่ยากระดับกลาง ผู้ใช้ทั่วไปควรจะจำได้ประมาณ 90% ของการ์ดที่โตเต็มที่เวลาทบทวน  
คุณสามารถตรวจสอบประสิทธิภาพตัวเองได้จากกราฟสถิติของเด็ค โดยดูที่กราฟปุ่มตอบคำถาม (Answer Buttons) – ค่า “mature retention” คือเปอร์เซ็นต์ถูกต้องที่แสดงด้านขวาของกราฟ  
ถ้าคุณเพิ่งเริ่มเรียน อาจยังไม่มีการ์ดโตเต็มที่เลย  
เนื่องจากประสิทธิภาพกับการ์ดใหม่และการ์ดที่ยังไม่โตเต็มที่อาจแตกต่างกันมาก จึงควรรอให้มีการทบทวนการ์ดโตเต็มที่จำนวนพอสมควรก่อนจะสรุปเรื่องอัตราการคงทนของตัวเอง

ในเว็บไซต์ SuperMemo แนะนำให้หาค่าตัวคูณที่เหมาะสมสำหรับอัตราการคงทนที่ต้องการ  
สูตรโดยสรุปคือ:

    log(เปอร์เซ็นต์การคงทนที่ต้องการ) / log(เปอร์เซ็นต์การคงทนปัจจุบัน)

สมมติว่าปัจจุบันคุณมีการคงทน 85% และต้องการเพิ่มเป็น 90%  
ตัวคูณจะคำนวณได้เป็น:

    log(90%) / log(85%) = 0.65

คุณสามารถใช้ [Google คำนวณได้](https://www.google.com/search?q=log(90%25)+%2F+log(85%25))

ถ้าคุณใส่ค่า 65% ลงในตัวคูณช่วงเวลา คุณจะพบว่าเวลาผ่านไป การคงทนจะใกล้เคียงกับค่าที่คุณต้องการมากขึ้น

อย่างไรก็ตาม สิ่งสำคัญคือ การแลกเปลี่ยนระหว่างเวลาศึกษากับการคงทนไม่เป็นเชิงเส้น:  
เพื่อเพิ่มการคงทนขึ้น 5% คุณต้องทบทวนบ่อยขึ้นถึง 35%  
ถ้าเนื้อหาที่เรียนสำคัญมาก อาจคุ้มค่ากับความพยายามนี้ แต่เป็นสิ่งที่คุณต้องตัดสินใจเอง  
ถ้าคุณแค่กังวลว่าลืมมากเกินไป อาจลองใช้เวลามากขึ้นในขั้นตอนเรียนรู้ตอนแรก หรือใช้เทคนิคช่วยจำ (mnemonics) จะได้ผลดีขึ้นในเวลาน้อยกว่า

สิ่งสุดท้ายที่ควรทราบคือ Anki บังคับให้ช่วงเวลาทบทวนใหม่ต้องนานกว่าช่วงเวลาก่อนหน้าขั้นต่ำ 1 วัน  
เพื่อไม่ให้ติดอยู่กับการทบทวนช่วงเวลาเดิมๆ ตลอดไป  
ถ้าคุณต้องการทบทวนการ์ดวันละครั้งหลายวันติดต่อกัน  
ควรตั้งขั้นตอนเรียนรู้เพิ่มแทนที่จะปรับตัวคูณนี้

### ช่วงเวลาปุ่ม Hard (Hard Interval)

ตัวคูณที่ใช้เมื่อกดปุ่ม **Hard**  
เปอร์เซ็นต์นี้สัมพันธ์กับช่วงเวลาก่อนหน้า เช่น ค่าปริยาย 1.20 หมายความว่าการ์ดที่มีช่วงเวลา 10 วัน จะถูกตั้งเป็น 12 วัน

### ช่วงเวลาปุ่ม Again (New Interval)

ตัวคูณที่ใช้เมื่อกดปุ่ม **Again** บนการ์ดทบทวน  
ค่าปริยาย 0.00 หมายความว่าช่วงเวลาของการ์ดทบทวนจะถูกรีเซ็ตเป็นศูนย์เมื่อคุณลืม  
(ซึ่งจะกลายเป็น 1 วันหลังจากใช้ [Minimum Interval](#minimum-interval))

ถ้าตั้งค่าแตกต่างจากปริยาย การ์ดที่ลืมแล้วอาจรักษาส่วนหนึ่งของช่วงเวลาก่อนหน้าไว้ได้  
เช่น ถ้าการ์ดมีช่วงเวลา 100 วัน และตั้งค่า **New Interval** เป็น 0.20  
ช่วงเวลาใหม่จะเป็น 20 วัน

แม้การรักษาส่วนหนึ่งของช่วงเวลาอาจดูสมเหตุสมผล  
SuperMemo พบว่าการรักษาช่วงเวลาไว้บางส่วนอาจ [ส่งผลเสีย](https://supermemo.guru/wiki/Post-lapse_stability)  
ด้วยเหตุนี้ เราแนะนำให้ใช้ค่าปริยาย

```javascript
// แสดงสถานะปัจจุบัน
console.log(
  JSON.stringify(states, null, 4)
);

// โหลดดีบักเกอร์ถ้าเปิด web inspector
debugger;

// ถ้าปุ่ม hard เป็นขั้นตอนเรียนรู้
// ตั้งเวลาช้าลงเป็น 123 นาที
if (states.hard.normal?.learning) {
  states.hard.normal.learning.scheduledSecs = 123 * 60;
}

// ใช้การเปลี่ยนแปลงเดียวกันใน filtered deck
if (states.hard.filtered?.rescheduling?.originalState?.learning) {
  states.hard.filtered.rescheduling.originalState.learning.scheduledSecs =
    123 * 60;
}

// เพิ่ม ease factor 0.2 เมื่อใช้ Easy ในการทบทวน
if (states.good.normal?.review) {
  states.easy.normal.review.easeFactor =
    states.good.normal.review.easeFactor + 0.2;
}
```

คุณยังสามารถดู [ตัวอย่างโค้ดตั้งเวลาแบบกำหนดเองของ FSRS](https://github.com/open-spaced-repetition/fsrs4anki/blob/main/fsrs4anki_scheduler.js) ได้

สถานะการตั้งเวลาต่าง ๆ ของการ์ดถูกอธิบายไว้ใน [SchedulingStates](https://github.com/ankitects/anki/blob/main/proto/anki/scheduler.proto)
